# **PRD: 程序化盘后流动性与尾部风险监控引擎 (V1.1)**

## **1\. 产品愿景与目标**

本项目旨在构建一个自动化的本地数据管道（隶属于 alphaear-logic-visualizer 架构下的宏观与衍生品监控模块）。系统在每日盘后自动抓取、清洗并计算关键的宏观流动性与期权衍生品指标。通过程序化扫描核心宽基 ETF 的隐含波动率斜率及保证金债务动量，为后续的量化策略或主观资产配置提供右侧“尾部风险”预警信号。

## **2\. 核心架构设计**

系统延续标准的**解耦三层处理流水线**（Decoupled Three-Layer Pipeline）架构：

* **数据接入层 (Data Ingestion Layer):** 负责 API 批量轮询、速率限制控制（Rate Limiting）与多标的原始 EOD JSON/CSV 数据的本地化存储。  
* **计算引擎层 (Calculation Engine Layer):** 基于 Python 运行环境，负责清洗多标的日终数据帧（EOD Dataframes），执行 Delta 插值与 Skew 矩阵计算。  
* **展现与预警层 (Presentation & Alert Layer):** 将标准化的计算结果生成 API 负载（Payloads），推送到基于 Google Apps Script 的 Google Sheets 日常宏观风险看板，或触发本地终端警报。

## **3\. 功能模块详细说明**

### **3.1 模块一：核心指数 ETF 期权暗流扫描 (Daily EOD)**

**目标：** 每日盘后批量分析美股核心宽基 ETF 的期权链，提取并横向对比各板块资金对下行保护的定价差异。

* **监控标的池 (Target Ticker Pool):**  
  * **SPY** (标普 500 ETF \- 衡量全市场大盘基准)  
  * **QQQ** (纳斯达克 100 ETF \- 衡量科技巨头与成长股风险)  
  * **IWM** (罗素 2000 ETF \- 衡量中小盘流动性与实体经济信用压力)  
  * **DIA** (道琼斯工业平均指数 ETF \- 衡量传统蓝筹股)  
* **数据源映射：** FMP API / Polygon.io (获取上述标的的日终期权链), VIX Central / Cboe (获取 VIX、VXN 历史数据)。  
* **核心计算逻辑 (按标的循环执行):**  
  * **固定 Delta IV Skew 矩阵计算:**  
    * 提取目标 ETF 距离到期日（DTE）约 30 天的期权链。  
    * 通过样条插值（Spline Interpolation）精准定位 $25\\Delta$ 的 Put 和 $25\\Delta$ 的 Call。  
    * 计算公式：$Skew\_{spread\\\_ticker} \= IV\_{Put, 25\\Delta} \- IV\_{Call, 25\\Delta}$  
  * **跨标的 Skew 剪刀差 (Cross-Asset Skew Spread):**  
    * 计算 $Skew\_{QQQ} \- Skew\_{SPY}$：当该差值急剧走阔时，预示科技巨头面临极端的去杠杆压力。  
  * **波动率期限结构 (Term Structure):**  
    * 对比近月与次月 VIX (针对 SPY) 和 VXN (针对 QQQ) 期货合约结算价，判断是否发生倒挂。  
* **预警触发条件（示例）：**  
  * 某单一标的 $Skew\_{spread}$ 突破过去 90 个交易日均值的 \+2 个标准差。  
  * QQQ 的 IV Skew 相对 SPY 出现统计学意义上的显著异常（资金集中抛售科技股保护）。

### **3.2 模块二：宏观流动性与杠杆压力测试 (Monthly)**

**目标：** 追踪 FINRA 保证金债务与 FRED 货币供应量的背离，寻找宏观顶部的确认信号。

* **数据源映射：** FRED API (获取 M2 数据), FINRA 官网自动化抓取或手工上传。  
* **核心计算逻辑：**  
  * **杠杆极限占比:** $Ratio \= \\frac{Margin Debt}{M2 Supply}$  
  * **动量反转计算 (MoM/YoY):** 动态计算保证金债务总额的同比与环比变化。  
* **预警触发条件（示例）：**  
  * 绝对值占比 $Ratio \> 6.0\\%$，且绝对值连续两个月环比萎缩或同比转负。

### **3.3 模块三：数据清洗与输出管道**

* **数据验证过滤器 (Data-validation Filters):**  
  * 剔除期权链中 Bid/Ask 价差异常（如无流动性的远期深度虚值期权）的脏数据。  
* **负载与展现 (Payload & Dashboard):**  
  * 将各 ETF 的 Skew 数值打包成 JSON 格式。  
  * 自动推送到 Google Sheets 风险看板，按列生成 SPY、QQQ、IWM 的 Skew 历史走势曲线。

## **4\. 最小可行性产品 (MVP) 开发路线图调整**

**Phase 1: 多标的数据管道连通 (Week 1\)**

* 配置 Python 环境。  
* 编写 API 轮询脚本，能够连续抓取 SPY, QQQ, IWM, DIA 的 EOD 期权链数据。  
* 处理 API 请求的并发与错误重试机制。

**Phase 2: 插值算法与矩阵对比 (Week 2\)**

* 编写 Python 函数实现精准的 $25\\Delta$ IV 提取。  
* 将所有标的的 Skew 计算结果汇总到一个主数据帧 (Master Dataframe) 中，生成每日截面数据。

**Phase 3: 自动化看板对接 (Week 3\)**

* 调试基于 Google Apps Script 的 Webhook，确保 Python 后端推送的多标的 JSON 负载能够正确解析并更新到 Google Sheets 的对应单元格中。
